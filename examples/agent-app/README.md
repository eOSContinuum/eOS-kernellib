# agent-app: the agent identity worked example

An agent authenticates through the key ceremony and exercises a
delegated capability end to end, entirely against the platform's
public surfaces. The example is the worked binding of the agent
doctrine (`docs/identity.md` Agent identities; delegation doctrine in
`docs/capability.md` Identity principals).

What the boot-time driver (`sys/test.c`) proves, self-contained at
tier E:

- a **controller identity** registered through the `authd` facade with
  a foreign-generated WebAuthn vector (attestation `"none"`, so the
  registration vector needs no signature);
- **controller self-service minting**: the agent's controller edge is
  derived from the proven session, never supplied;
- the **key ceremony** with a foreign-signed, domain-separated
  assertion (`sys/vectors.h`, generated by
  `scripts/gen-agent-vectors.py`; Ed25519 signatures from the openssl
  CLI), plus the two refusals that make it meaningful: a signature
  over the bare challenge (the domain tag is load-bearing) and a
  wrong-key signature;
- the **refusals the doctrine requires before any operator action**:
  delegation refuses on the live-hold check, and the application's
  action gate (an `is_allowed` check on `agentapp.write`) stays
  closed;
- **suspension** blocking the ceremony and killing the live session,
  and **resume** restoring authentication.

Granting the controller the capability and flagging it delegable are
deliberately operator actions the example cannot perform itself. The
operator continuation -- `scripts/verbsets/agent-app.verbset`, which
grants, flips the delegable flag, and calls the driver's
`continue_with_operator_grant()` -- proves the second half: the
delegation lands, the agent's principal passes the action gate,
suspension kills the delegated grant, and resume restores nothing.

## Verify

Both runs need the crypto module (`LPC_EXT_CRYPTO`) and a DGD binary
(`DGD_BIN`).

Boot phases only (10 sentinels):

    LPC_EXT_CRYPTO=/path/to/crypto.<ver> DGD_BIN=/path/to/dgd \
        scripts/run-example.sh agent-app

End to end including the operator continuation (the verbset asserts
the four continuation sentinels and the absence of FAIL lines):

    LPC_EXT_CRYPTO=/path/to/crypto.<ver> DGD_BIN=/path/to/dgd \
        DEPLOY="agent-app:AgentApp" scripts/drive-verbs-smoke.sh \
        scripts/verbsets/agent-app.verbset

Sentinels land in `/usr/AgentApp/data/test-result.log` (the
application-tier logging convention shared by the bundled examples).

Regenerating the vectors (fresh Ed25519 keypair; fixed structures):

    python3 scripts/gen-agent-vectors.py
